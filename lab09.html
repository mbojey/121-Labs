This lab will focus on implementing a sorting algorithm called merge sort. You can read about it here

Exercise 1
[1 point] In this lab you will be writing a recusive method. To become familiar with these types of methods trace through the following code with n = 6. At each iteration record what the current value of total is.

public static int total = 1;
public static int factorial(int n){

if(n == 0 || n == 1)

return 1;

total = n * factorial(n-1);

return total;

}

The first time that the method is called total would be equal to 6 * factorial(5). Then in the call to factorial(5) total would be equal to 5 * factorial(4). As these calls finish total would be equals to 5 * 24 = 120 and 6 * 120 = 720. Trace through the rest of the code on your own.

Exercise 2
[1 point] How many times does mergesort need to "split apart" an array of size n? Remember that we split the array in half every time and stop when there is only one element in the smaller parts.

Lab Activity - 8 points

What you are given:

  • Your work from the previous labs

What you have to do:

  • Today you will be writing your own code to perform a merge sort.
  • [1 point] Start by making a merge sort class, call it MergeSort.java. This class will have two int array attributes, one called values and one called temp. The constructor will take no parameters.
  • [1 point] Make a method called sort() that takes an int array as a parameter. Set values equal to this input array and set temp to an array of the same size. Now call a method mergesort(), which we will fill in next, with 0 and the size of the array values as the 2 int parameters.
  • [2 points] Make the mergesort() method, it will take parameters as described above. Check if the first parameter is smaller than the second, if it is then that means that there is still elements in the array between them. Calculate the middle of the two parameters and store the value in a variable. Now recursively call mergesort() twice, once with the small parameter and the middle value and once with the middle value and the large parameter. This recursive call will seperate each side of the array until there is only one element in each subarray. Now call a new method, merge(), which we will fill in next, with the smaller parameter, the middle value, and the larger parameter.
  • [2 points] Make the merge() method, it will take parameters as described above. Start by copying all values from the indicies between the smallest parameter and the largest parameter from values to temp. Declare 2 variables and initialize them to the smallest parameter, also declare 1 variable and initialize it to the middle parameter + 1. Now, copy the smallest value from either the left or right side of the middle parameter back into values. Make sure you stop when you run out of values on either side. Finally, copy the remaining values from the side that didn't run out of values into values
  • [1 point] Write comments for the methods you just wrote in your own words. Your TA should be able to just read your comments and know what the method does at each step.
  • [1 point] Now you are done! Write a test class to test your implementation. Give the class an array of 10 random integers and print out the result to see if it worked. It may not work the first time, but that is fine, just ask your TA for help! Save your output into a file called output.txt

What you should think about during this activity:

  • What other types of algorithms could take advantage of this "divide and conquer" approach?
  • Why is efficiency so important in sorting algorithms?
