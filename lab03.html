Lab 3

Objectives

After completing this lab you will be able to:

  • Write an interface given an English description of the requirements.
  • Be able to implement an interface in your code.

Pre-Lab Exercises - 4 points

Complete this section individually before beginning the lab activities. This section will be due AT THE TIME YOUR LAB SECTION STARTS.

Exercise 1
[2 points] In lecture and in your readings (pp 323 - 329) you have seen examples of interfaces. Given the following description of what all classes implement an interface are required to do create an interface called PreLab.java. Write the interface as if it would be used in a project, i.e. use proper syntax.

Classes that implement the interface you create must have methods to access and mutate a String variable called name and an integer variable named count. Classes must also have a toString() method that returns name and count as a String. Finally, classes that implement the interface must have a method called length() that returns an integer value and take a String as a parameter.

[2 points] Make 2 multiple choice questions that review concepts about interfaces. These questions should have 5 possible answers, 1 correct and 4 incorrect answers. Use the site here

Hand in your answers before your lab section starts.

Lab Activity - 8 points

What you are given:

  • Your work from previous labs

What you have to do:

  • In lecture you have examples of interfaces and seen the advantages it gives, such as enforcing classes to have certain methods. Today we will be making changes to the FeedMe game to take advantage of this fact so that we can enforce that all items that move on the screen, called Sprites, share the same methods.
  • [1 point] Rename the current Sprite class, call it Monster.
  • [1 point]Create a new interface and call it Sprite. You create interfaces the same way as you create classes in Eclipse. Just click Interface instead of Class.
  • [2 points]Make a list of all methods common to BOTH the Fruit and Monster classes. Write this list in a .txt file called similarities.txt
  • [2 points]Add all of these methods to the Sprite interface.
  • [2 points]Make the Monster and Fruit classes use the new Sprite interface. This is done with the use of the implements keyword. Make sure that you have no errors in your project.
  • All done! Now you know that our Monster and Fruit classes will have to have the methods you declared in the interface.

What you should think about during this activity:

  • What are the other advantages of interfaces?
  • What are some possible disadvantages of interfaces?
  • Why do we need to have no method bodies in interfaces?
  • We didn't use any constants in our interface. If we had, they would need to have been static. Why are constants static in interfaces?
  • How are interfaces differeent from inheritance?

What to Submit

  • Your Sprite.java file
