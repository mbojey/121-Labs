Lab 7

Objectives

After completing this lab you will be able to:

  • Write an array based Queue.
  • Write a list based Stack.

Pre-Lab Exercises - 4 points

Complete this section before beginning the lab activities. This section will be due AT THE TIME YOUR LAB SECTION STARTS.

This lab will focus on implementing a stack using the doubly linked list from last lab. You will also make a queue using an array.

Exercise 1
[1 point] Explain, in your own words, what is the difference between a stack and a queue.
Exercise 2
[1 point] What is one instance where you think using a stack would be beneficial? Why? Also, what is one instance where you think using a queue would be beneficial? Why?

[2 points] Make 2 multiple choice questions that review concepts from stacks and queues. These questions should have 5 possible answers, 1 correct and 4 incorrect answers. Use the site here

Lab Activity - 8 points

What you are given:

  • Your work from the previous labs

What you have to do:

  • Today you will be making your own stack and queue classes. The underlying data structure to be used in the stack is the doubly linked list you made last week. We will use an array for the queue.
  • Start by creating a new class, call it ListStack. This class will have all the methods of the stack Abstract Data Type(ADT), that is push() pop() peek() isEmpty(), we will also have a size() method.
  • [1 point] In the ListStack class you will need an attribute of type List. Write a constructor to initialize this attribute. Also write the size() and isEmpty() methods now.
  • [1 point] Now fill in the push() and pop() methods. Make sure that you are passing the proper parameters and return the correct data. These methods will be very easy with some of the method you wrote in your List class last week.
  • [1 point] Write the peek() method.
  • [1 point] You have now finished all the methods required for the stack ADT. Now make a new class ArrayQueue. This class will need to have three methods to be a queue, enqueue() dequeue() isEmpty(), we will also have a size() method. Declare two int attributes to keep track of the front and back of the queue, initialize these in the constructor. This class will also need to have an array attribute that can hold Fruit. In the constructor initialize this array to whatever size is passed as a parameter. Also write a toString() method to print out the contents of the queue.
  • [1 point] Start by writing the enqueue method. In this method you will add a Fruit to the back of the queue. After you have added the Fruit move the attribute that keeps track of the back of the queue. Be careful, you need to make sure that you do not go out of bounds on your array.
  • [1 point]Now write the dequeue method to remove the first Fruit from your array. When you remove the Fruit set the value stored in that index of the array to null. Now increment you front attribute, again be careful to no go out of bounds.
  • [2 points]Write the size() and isEmpty() methods. Notice that these are more difficult than they may first appear, be caeful to cover all cases. When these methods are complete add printed warnings to enqueue and dequeue telling the user when the queue is full or empty.
  • [1 point] Write a test class for each of the class you just made. Make sure to test all methods and all cases when testing the queue.

What you should think about during this activity:

  • What advantages does a list have over an array?
  • Do you think using an array or a list is easier when implementing queues and stacks? What are the advantages of each?
