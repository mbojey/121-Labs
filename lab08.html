Lab 8

Objectives

After completing this lab you will be able to:

  • Search for an item in a list using a linear search.
  • Search for an item in a sorted collection using a binary search.

Lab Activity - 8 points

What you are given:

  • Your work from the previous labs

What you have to do:

  • Today you will be writing your own code to perform a linear search and a binary search. The code you write for the linear search will be used in the game and the code you write for the binary search will be great practice.
  • [1 point] The first thing we will do is to search through all the Fruit dropped in a game and determine if each type of Fruit you have made is dropped in a single round. To do this, make a method called usedFruits() in the Drop.java class and call it whenever a fruit is dropped.
  • [1 point] In this method, you will search through the list of Fruits used in the game and record if each type of Fruit was used. To store this information make a boolean attribute in the Drop.java class for each type of Fruit in your game. Initialize these attributes in the constructor and write accessors(getters) for them.
  • [1 point] Now get the list of Fruit used in the game. Move through this list once for each type of Fruit in your game. If you find a Fruit of that type, set the value of the attribute to true. Remember, this is a doubly linked list so you can move through the list using the next attribute at each node. To start at the front of your list you will need to write an accessor for the front attribute.
  • [1 point] In paint() when the game is over we want to let the player know what types of Fruit they found in the game. For each boolean Fruit attribute write a statement that tells the player if they used that type in the current round. Pay attention to spacing, if you have a large variety of Fruit you may want to use a smaller font.
  • [1 point] Now we will move on to writing code for a binary search. To do this make a new class called BinarySearch. This class will need to have two int arrays, one called temp and the other original. Initialize these attributes in the constructor. When you initialize the array, call the Arrays.sort() method with the array as the parameter. You will need to import the java.util.Arrays class to do this. This is because binary search only works on sorted collections. You will write you own sort method next lab.
  • [2 points] Write a public method called search that takes an int parameter and returns a boolean value. This method will search through the array to find the given value. To start, call a method to find the half way point of your array, if there is an even number of indicies in your array just round down. Now compare the value at the halfway point of temp. If it is less than the value, use the Arrays.copyofRange() method to reassign temp to only it's second half and then recursively call search. Do a similar procedure if the value if greater, only this time use the lower half of temp. If the halfway point equals the value return true. The final case is that temp is only 1 entry, if this is true and the value does not equal that one entry, return false. When the search is done, set temp equal to original.
  • [1 point] Finally write a test class for your binary search to make sure that it works. Submit the output from this class as well as the class.

What you should think about during this activity:

  • What advantages does a linear serach have over a binary search?
  • What other types of searching algorithms do you know of or can think of?
  • Why is efficiency so important in searching algorithms?
